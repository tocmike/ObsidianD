# 如何用 AI 把语音录入文本转成自己独特风格？

这篇内容，是「内容录入与整理」系列文章的第三部分。

第一篇是《如何用语音记临时笔记？》我为你介绍了免费而高效的语音录入转文字方法。

![[./_resources/如何用_AI_把语音录入文本转成自己独特风格？.resources/e1a383c15a6781257461623d053eeed6.webp]]

但是，仅仅转换录入语音，显然无法直接用来输出，你需要做整理。所以在第二篇《GPT-3 加持的语音输入文字新流程》一文中，我给你介绍了如何用 GPT-3 来整理语音转文字稿件，把口语化文字转成书面化，减轻手工整理的负担。这里你可以使用几种不同的工具，包括 Notion AI 和 Craft AI assistant 等，因为它们背后调用的 API 其实大同小异。

复习一下，你只需要使用这样的提示语（prompt），就可以把杂乱的语音识别文稿，变成文从字顺的版本。

「用简洁的语言整理这一段话，要逻辑清晰，去掉错别字。」

![[./_resources/如何用_AI_把语音录入文本转成自己独特风格？.resources/d633b7e3f36f82c60d8396e688eeeec4.webp]]

转换之后的结果，就成了这样子：

![[./_resources/如何用_AI_把语音录入文本转成自己独特风格？.resources/d633b7e3f36f82c60d8396e688eeeec4.webp]]

这里的关键，在于提示语 prompt 的撰写。这其实已经成了一门显学。如果你有意学习的话，这里有个不错的免费教程，适合循序渐进理解和掌握 prompt 的编写精髓。

![[./_resources/如何用_AI_把语音录入文本转成自己独特风格？.resources/b355332296e73a4a774ff0d573436742.webp]]

注意网站作者非常细心地对不同学习材料进行了难度评级。绿色代表最容易，甚至不需要有任何基础。你可以先把容易的部分学完，观其大略。

在文本整理这个任务里，最让人头疼的也恰恰是 prompt 的撰写。在上一篇文章里，咱们尝试明确指定想要转换目标文本的风格。可惜这个过程并不总是一帆风顺，你会经常遇到挑战。最大的问题在于，AI 并不是你「肚子里的蛔虫」，它无法通过猜测来准确了解你用简单语言做的「叮嘱」。

好比说，你要求「简洁」，那些口语化的口癖类文字确实会被去掉，不少短句也可以合并在一起。从这个角度来讲，GPT-3 确实很厉害。问题是 AI 时常自作聪明，因为你提出了「简洁」要求，就大胆帮你把文本进行「缩句」。原来充实饱满的语言，一下子肉都没了，只剩下骨架。转写任务突然变成了「提炼主要内容」，不符合咱们实际的转写需求。

我前些日子对 AI 的自作主张大为头痛，没少进行各种测试。一会儿增加一些要求，一会儿再调整一下文字。Craft 提供的 AI assistant 额度快速消耗，看着真让人心疼啊。后来，我的提示语写得都比转换的段落长了，效果却还不够理想。另外在这样 prompt 要求下 AI 写出来的文字，与我自己的写作风格差异较大，后续也少不了要进行大幅度的修改，想起来就头疼。如果你工作中时常改文章，可能会有共鸣 —— 改别人的文章，可比自己从头写起来还要费劲许多。

昨天一个偶然的机会，我突然发现 ChatGPT 可以帮助咱们更高效处理这个问题。

其实说起来，ChatGPT 的基础也是 GPT-3 ，只不过用于问答任务。它似乎没有道理会比 Craft 的 AI assistant 或者 Notion AI 更好用。但是你别忘了， ChatGPT 比起这些简单集成在写作工具里的 AI 有两个特性：

首先，是 ChatGPT 采用对话方式，所以支持迭代改进。你可以根据输出的结果，只去直接指明需要添加、删除、提升和降低风格要素，看改写的结果是否符合需求，从而快速改进。这个过程里，你不需要完全重述 prompt ，只需要输入变化部分即可，更有利于对要求进行微调。一旦确认结果符合需要，你还可以让 ChatGPT 自行总结完整的 prompt ，便于日后一站式使用。我在这则视频里面给你演示过相关的技巧，可以点击复习。

第二，是更好玩儿的 —— 你可以利用写过的现成文本，喂给 ChatGPT 让它「自己揣摩」。我是在使用 ChatGPT 编写代码处理文本替换的时候，偶然发现 ChatGPT 可以按照模糊的指令加实际的例子，自己学习规律。这实在是好玩儿。因为「百闻不如一见」嘛，照葫芦画瓢比起「抽象总结」，要实用多了。

于是我拿出最近的一期公众号文章稿件，稍作调整去掉了那些图片引用的 Markdown 语法等干扰因素，就把正文扔给了 ChatGPT。当然，你得在文稿前面加上一句提示语：

「总结以下文字的风格，加入一点幽默感，并将该风格保存为 "wsy"。」

![[./_resources/如何用_AI_把语音录入文本转成自己独特风格？.resources/a0012d8ff0fb96a82a273079e23b582b.webp]]

这里得给你一个小提示 —— 因为训练语料比例的差异，使用 prompt （提示语）时最好用英文来写。不是咱崇洋媚外，主要是 ChatGPT 这东西食谱不均衡，它的胃口更适应英文，处理起来更加得心应手。

![[./_resources/如何用_AI_把语音录入文本转成自己独特风格？.resources/d5ea4119ec585b33c3b06fc2ffa0a387.webp]]

ChatGPT 总结出来的文本风格特点，让我感觉很受用，甚至感觉它有意在夸赞我。这些风格描述内容总结成文字显得有些单薄，不够具象化。不过模型体会到的风格是由海量参数调整来记忆的，所以即便「茶壶倒饺子」表达不出来，模型应该也已经能体会和揣摩到了。

下面，我就把自己语音输入的内容，扔了进来。注意这次前面加上提示语：「用 "wsy " 文体改写以下文字。」你看，文体已经成了一个变量。前面的工作是给变量赋值，现在可以随时调用它了。

![[./_resources/如何用_AI_把语音录入文本转成自己独特风格？.resources/ddc0774c10f2caf5103d970f617bc08c.webp]]

由于咱们的目标是提升录入效率，输入内容是语音识别的口语化文本。其特点是想到哪里说到哪里，逻辑难免会有不通顺的地方。另外文字识别准确度注定到不了100%，大都有识别错误的情况。不过，你可以看到ChatGPT尽心尽力帮助我们扫除了不少重复冗余和语病，并且尽量理顺了逻辑。特别是，它还自动帮助咱们把语音识别错误的英文词汇 "chat jpt" 或者 "Chat gBT" 都统一变成了 ChatGPT 。这确实需要 AI 对上下文足够的把握能力，才能做到。看到一个 AI 帮助另一个 AI 纠错，我很惊喜。

上面的例子里「加入幽默」是我反复尝试以后调整出来的 prompt 。如果你只让 ChatGPT 学习语言风格，它学习的结果总是「过于严肃」。我觉得这可能与 ChatGPT 的默认设置有关。ChatGPT 上面被人为加上了很多限制，甚至许多问题它都避而不答，这早就不是什么秘密了。为什么 ChatGPT 把「严肃」等级调那么高？我没有看过限制条件的源码，所以不能确定。但我的估计是 OpenAI 不想让产品的回答显得轻浮。这种深度交互产品的应用中，用户的信任很重要。如果 ChatGPT 给出的答案嬉皮笑脸，对用户的信任不是加分项。既然如此，咱们自己矫枉过正吧，让它加一些幽默元素。

不过注意「过犹不及」，万事都得掌握一个「度」。如果你添加幽默风格太多，ChatGPT 的回答就会变成这个样子：

![[./_resources/如何用_AI_把语音录入文本转成自己独特风格？.resources/250b9f227e5618b2faf8cdd61147ad8c.webp]]

确实太油嘴滑舌了，还是调整回来一些好了。

我用同样的方式，处理了全部语音输入的段落。这个过程里，因为每次输入，都需要拷贝两种内容（一个提示语，一份要撰写的文字），觉得很麻烦。于是我还随手编了一个 Keyboard Maestro 的宏命令，帮助我用快捷键一次性输入提示语和文本。方便多了。

## 小结

本文咱们谈了如何用 ChatGPT 自动提炼和模仿作者的语言风格，将语音输入转换出来的原始口语文本，转换成更为实用的书面语言，以便后续的调整、发布和复用。希望这次的讲解，有助于提升你的文本输入效率，在更短的时间里，写出更多好文章。祝写作愉快！
